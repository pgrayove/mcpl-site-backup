Tests are single tests for accessibility that are provided by the QUAIL library. QUAIL supports all tests outlined by the Open Accessibility Tests standards; however, you can also write your own.

The Accessible Content module adds all the supported tests as a node to your Durpal site, so it is easy to edit their titles and descriptions. Site administrators might want to change the body of their tests so that when an author finds a problem, the test will tell them how to fix the issue in a way that is particular to their site.

Severity Levels

Tests also have severity levels which break down to the following:

  • Severe Quail is 100% certain that this is an error.
  • Moderate Quail has found a potential error that requires human interpretation.
  • Suggestion These are errors which might point to tips regarding language usage or the flow of a document that requires human interpretation.

Site administrators can change the severity level of any test; however, it is suggest that the default level be chosen.

Writing Custom Tests

You can write your own custom tests for QUAIL by following the Wiki page guidelines at the QUAIL Project Site.
